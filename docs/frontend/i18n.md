# Frontend Internationalization (i18n) Architecture

The `mango-ui` React frontend uses `i18next` and `react-i18next` for localization to support multiple languages (English and Vietnamese natively).

## Technical Stack
- **Core Library:** `i18next` (Handles dictionary loading and interpolation)
- **React Adapter:** `react-i18next` (Provides Context Providers and the `useTranslation` hook)

## File Structure
- `src/i18n.ts`: The main configuration file. Bootstraps the `i18next` instance and loads the static dictionaries.
- `src/locales/{lang}/translation.json`: The JSON dictionaries containing the translated key-value pairs.
- `src/components/Navbar/LanguageSelector.tsx`: The UI component responsible for toggling the active language context.

## Usage Guidelines

### 1. Adding New Translations
When adding new static text to the UI, do not hardcode strings. Instead, add them to **both** `en/translation.json` and `vi/translation.json` using logical nesting.

```json
// translation.json
{
  "cart": {
    "title": "Your Cart"
  }
}
```

### 2. Consuming Translations in Components
Use the `useTranslation` hook to access the `t` function. 

```tsx
import { useTranslation } from 'react-i18next';

export function CartPage() {
    const { t } = useTranslation();
    return <h1>{t('cart.title')}</h1>;
}
```

### 3. String Interpolation
For dynamic values, use Handlebars-style interpolation in the JSON, and pass the variables in the `t` function call.

```json
// translation.json
{
  "orders": {
    "itemCount": "{{count}} item(s)"
  }
}
```

```tsx
<span>{t('orders.itemCount', { count: 5 })}</span>
```

### 4. Language Switching Context
The language state is globally managed by `i18next` inside the React Context. Calling `i18n.changeLanguage('vi')` automatically triggers a re-render of all components utilizing the `useTranslation` hook.
